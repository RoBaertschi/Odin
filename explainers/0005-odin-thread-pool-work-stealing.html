Odin's Thread Pool Runs Tasks On Existing Workers

The checker submits procedure bodies as tasks. The thread pool does not spawn a new thread per task; it starts a fixed set of worker threads, gives every thread its own queue, and lets workers steal from one another.

The Loop

Init

Create worker_count + 1 threads. Slot 0 is the main thread.

Submit

thread_pool_add_task pushes a task onto the current thread's queue.

Local Work

A worker pops tasks from its own queue first.

Steal

If its queue is empty and work remains, it steals from another thread's queue.

Sleep

If no work is available, it waits on a futex until new work is broadcast.

One Queue Per Thread

Each Thread owns a TaskQueue with a ring buffer plus top and bottom indexes. The owner usually pushes and pops at the bottom. Other workers steal from the top.

Mental Model

WorkerTask is just a function pointer plus data pointer. For procedure-body checking, the function is check_proc_info_worker_proc and the data is a ProcInfo *.

typedef struct WorkerTask {
    WorkerTaskProc *do_work;
    void           *data;
} WorkerTask;

Submitting Work

Adding a task packages the function/data pair and pushes it to the current thread's queue. Pushing increments tasks_left and wakes sleepers if needed.

thread_pool_add_task(pool, proc, data):
    task.do_work = proc
    task.data = data
    push(current_thread.queue, task)
    tasks_left += 1
    wake sleeping workers

Running Work

Workers prefer their own queue. After each task, they decrement tasks_left. If their queue is empty but tasks_left says work exists somewhere, they scan other queues and steal one task.

worker loop:
    while own queue has task:
        run task
        tasks_left -= 1

    if tasks_left > 0:
        try stealing from other queues

    if nothing found:
        futex_wait(tasks_available)

Waiting Is Active Participation

thread_pool_wait is not a passive join. The waiting thread also drains its own queue while it waits for tasks_left to reach zero. In the checker, this means the main thread can execute body-checking tasks too.

while tasks_left != 0:
    while current_thread.queue has task:
        run task
        tasks_left -= 1
    futex_wait(tasks_left)

Important Correction

A task finishing does not start a new worker thread. A worker simply loops and takes another task, steals a task, or sleeps. Threads are created during pool initialization.
