Why Old Ring Reads Can Be Safe

The confusing part is that a stealer can temporarily see an old ring buffer while the owner has grown the queue. The safety comes from three separate facts: old rings stay alive, top/bottom are logical counters outside the ring, and bottom is the publish point for newly pushed work.

Step Through The Resize

Owner Thread View

top10
bottom14
ringold[4]

Stealer Thread View

loaded top10
loaded bottom14
loaded ringold[4]

old live task copied during resize new post-resize task old slot that must not win for new work

The Memory Ordering In Plain English

ring = new_ring
The owner publishes the new storage before exposing new work that needs it.
buffer[bottom] = task
The task is written into the current ring before the queue says it exists.
release fence
Prevents the ring/task writes from being observed after the bottom update.
bottom = bottom + 1
This is the publish point. A stealer only considers work available after seeing this.
acquire bottom load
If the stealer sees the new bottom, later reads must see the writes that were published before it.

The One Rule

A stealer may have an old ring pointer, but it can only win a task by advancing the shared logical top. If its view is stale and top moved, the compare-exchange fails and the task read is discarded.

Safe Old-Ring Case

stealer loaded old ring
stealer loaded old top = 10
owner grows and copies task 10 to new ring
stealer CAS top 10 -> 11 succeeds

Result:
  stealer owns logical task 10
  copied task 10 in new ring is ignored

The Bad Case Being Prevented

stealer sees new bottom
stealer uses old ring
stealer steals logical task pushed after resize

This would be wrong, because post-resize tasks
exist only in the new ring.

The acquire/release ordering around bottom is what prevents this successful combination.

Source Shape

// owner push path
if (queue_is_full) {
    queue.ring = grow_and_copy_old_live_tasks();
}

current_ring->buffer[bottom % current_ring->size] = task;
release_fence();
queue.bottom.store(bottom + 1);

// stealer path
top = queue.top.load();
bottom = queue.bottom.load(acquire);
if (top < bottom) {
    ring = queue.ring.load();
    task = ring->buffer[top % ring->size];
    if (CAS(queue.top, top, top + 1)) {
        run task;
    }
}
