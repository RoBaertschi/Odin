# Operands understood as checked expression results

The learner understands that an operand is a semantic result produced by expression checking, carrying a type, use classification, and possibly a compile-time constant value. They understand the practical C-compiler pattern of folding constant expressions opportunistically during expression checking: literals and named constants produce constant operands, and unary/binary operators check their child operands and produce a new constant operand when the operation is semantically valid and all required inputs are constant. Contexts such as array bounds, enum values, and case labels should then separately validate whether the constant is acceptable there. For the intended C compiler design, compile-time constant operands are rvalue-like semantic results; folding actual variables is backend optimization rather than semantic constantness.
